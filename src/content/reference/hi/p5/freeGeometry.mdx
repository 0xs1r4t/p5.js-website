description: >
  <p>Clears a <a href="/reference/p5/p5.Geometry">p5.Geometry</a> object from
  the graphics

  processing unit (GPU) memory.</p>

  <p><a href="/reference/p5/p5.Geometry">p5.Geometry</a> objects can contain
  lots of data

  about their vertices, surface normals, colors, and so on. Complex 3D shapes

  can use lots of memory which is a limited resource in many GPUs. Calling

  <code>freeGeometry()</code> can improve performance by freeing a

  <a href="/reference/p5/p5.Geometry">p5.Geometry</a> object’s resources from
  GPU memory.

  <code>freeGeometry()</code> works with <a
  href="/reference/p5/p5.Geometry">p5.Geometry</a> objects

  created with <a href="/reference/p5/beginGeometry">beginGeometry()</a> and

  <a href="/reference/p5/endGeometry">endGeometry()</a>,

  <a href="/reference/p5/buildGeometry">buildGeometry()</a>, and

  <a href="/reference/p5/loadModel">loadModel()</a>.</p>

  <p>The parameter, <code>geometry</code>, is the <a
  href="/reference/p5/p5.Geometry">p5.Geometry</a>

  object to be freed.</p>

  <p>Note: A <a href="/reference/p5/p5.Geometry">p5.Geometry</a> object can
  still be drawn

  after its resources are cleared from GPU memory. It may take longer to draw

  the first time it’s redrawn.</p>

  <p>Note: <code>freeGeometry()</code> can only be used in WebGL mode.</p>
